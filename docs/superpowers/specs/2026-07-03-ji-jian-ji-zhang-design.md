> 创建日期: 2026-07-03

## 一、产品定位

**产品名（暂定）**：随便记

**一句话定位**：跟聊天一样记账，AI 帮你分类分析。

**目标用户**：大学生（18-22岁），生活费有限、有记账意愿但没耐心、对新鲜事物接受度高。

**产品目标**：获得真实用户使用，跑出可量化的数据指标（DAU、周留存率、日均记账笔数），用于产品经理岗位面试作品集。

---

## 二、核心流程（3步闭环）

```
打开小程序
  → 说/写一句「食堂午饭刷卡12块」
  → AI 自动归类为「餐饮-食堂」，存好
  → 每周日20:00 推送周报：「这周你喝了6杯奶茶，花了83元💸」
```

## 三、MVP 功能范围

### P0 — 必须有，否则产品不成立

| 功能 | 描述 |
|------|------|
| 微信登录 | 一键授权，获取头像昵称 |
| 文本记账 | 输入框 + 发送，自然语言一句话记账 |
| AI 自动分类 | 自动识别金额 + 分类（餐饮/交通/购物/娱乐/学习/日用/其他），无需手动选 |
| 收支列表 | 按时间倒序展示，今天/昨天/本周分组 |
| 简单统计卡片 | 本月总支出、总笔数、Top类目排名及占比 |

### P1 — 上线后第一波迭代

| 功能 | 描述 |
|------|------|
| 语音记账 | 按住说话，语音转文字后走同样AI分类流程 |
| 周报推送 | 每周日20:00 微信模版消息推送 |
| 基础筛选 | 按类目/时间筛选账单 |

### P2 — 验证核心流程后再加

| 功能 | 描述 |
|------|------|
| 省钱战绩海报 | 可分享的周报图片生成 |
| AI 趣味洞察 | 「奶茶预警」「月光指数」「食堂性价比之王」等 |
| 快捷记账模板 | 预设常用模板，点击即记 |

### MVP 明确不做

- 预算设置
- 多账本/多人协同
- 账单照片 OCR
- 复杂图表分析页
- 导出账单

## 四、技术架构

```
┌─────────────────────────────────────┐
│         微信小程序前端               │
│    (原生微信小程序框架)              │
│    · 登录授权                        │
│    · 记账输入 + 语音                 │
│    · 列表 & 统计卡片                 │
└──────────────┬──────────────────────┘
               │ HTTPS
┌──────────────▼──────────────────────┐
│          后端服务 (Node.js + Express) │
│    · 用户管理                        │
│    · 账单 CRUD                       │
│    · 统计计算                        │
│    · 微信模版消息推送                 │
└──────────────┬──────────────────────┘
               │ API 调用
┌──────────────▼──────────────────────┐
│        AI 服务 (LLM API)             │
│    · 自然语言解析 → 金额+分类        │
│    · 周报文案生成                    │
└─────────────────────────────────────┘
```

### 技术选型

| 层 | 选型 | 理由 |
|---|------|------|
| 小程序 | 原生微信小程序 | 生态最佳、API 权限最全、包最小 |
| 后端 | Node.js + Express | 轻量、全栈统一语言、部署简单 |
| 数据库 | SQLite（MVP期） | 零配置、够用、后续平滑迁移 PostgreSQL |
| AI | Claude API / 性价比模型 | 自然语言解析刚需 |
| 部署 | 腾讯云轻量服务器 | 国内访问快、微信生态兼容好 |

## 五、数据模型

### 用户表 `users`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 用户唯一ID |
| openid | TEXT UNIQUE | 微信 openid |
| nickname | TEXT | 微信昵称 |
| avatar | TEXT | 头像 URL |
| created_at | DATETIME | 注册时间 |

### 账单表 `bills`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 账单ID |
| user_id | INTEGER FK | 所属用户 |
| type | TEXT | 'income' / 'expense' |
| amount | INTEGER | 金额，单位为分（避免浮点） |
| category | TEXT | AI 分类结果 |
| raw_text | TEXT | 用户原始输入，如「午饭食堂12块」 |
| recorded_at | DATE | 消费/收入发生日期 |
| created_at | DATETIME | 记录创建时间 |

### 分类枚举（7类）

```
餐饮  交通  购物  娱乐  学习  日用  收入
```

## 六、UI 设计

- 设计语言：iOS 风格，毛玻璃组件（backdrop-filter: blur），暖米色渐变背景
- 字体：Inter / SF Pro Display，紧凑字间距
- 组件：半透明白色卡片 + 柔光装饰 + 低饱和色彩体系
- 暗色模式：周报推送页采用深色背景 + 毛玻璃卡片
- Mockup 文件：`mockup-v2.html`

### 三个核心页面

1. **首页（记账）**：顶部问候卡片 + 今日支出统计，AI 分类气泡动画，账单列表按日期分组，底部常驻输入栏（文字 + 语音）
2. **统计页（月度概览）**：总支出大字展示，类目排名列表带进度条
3. **周报推送**：本周支出/笔数/TOP类目三栏，AI 洞察卡片（奶茶预警等），亮点总结，分享按钮

## 七、AI Prompt 设计

### 记账分类 Prompt

```
你是一个记账分类助手。用户会用自然语言描述一笔消费或收入，
你需要从中提取：金额、类型（支出/收入）、分类。

分类可选：餐饮、交通、购物、娱乐、学习、日用、收入

用户输入：「{raw_text}」

返回 JSON：
{
  "type": "expense" | "income",
  "amount": 数字（元，保留2位小数）,
  "category": "分类名",
  "note": "简短备注（如店名或物品名，可选）"
}
```

### 周报生成 Prompt

```
用户本周消费数据如下：
- 总支出：{total} 元
- 笔数：{count}
- TOP1 类目：{top_category}，占比 {pct}%
- 各分类明细：{category_breakdown}
- 与上周对比：{vs_last_week}

请生成一段温暖的、朋友式的周报摘要，包含：
1. 本周消费概况（1句话）
2. 一个有趣的洞察（如奶茶/外卖频率）
3. 一个省钱建议或鼓励

语气：轻松、朋友口吻、不爹味、偶尔用emoji
字数：150字以内
```

## 八、关键指标

上线后关注的数据：

| 指标 | 定义 | 目标（MVP首月） |
|------|------|----------------|
| DAU | 日活跃用户数 | ≥ 50 |
| 周留存率 | 首次使用后第7天仍活跃 | ≥ 30% |
| 日均记账笔数 | 每人每天平均记账次数 | ≥ 2.5 |
| AI 分类准确率 | 无需手动修改分类的比例 | ≥ 85% |
| 周报打开率 | 模版消息推送后的点击率 | ≥ 40% |

## 九、已知限制 & 后续方向

### 本版本限制
- 无预算管理功能
- 无多设备同步（仅微信环境）
- AI 分类依赖 LLM API，存在延迟和成本
- 微信小程序审核周期不可控

### 后续可扩展方向
- 支付宝/银行卡自动同步（高级版）
- 社交功能（好友比拼省钱、组队记账）
- 定制化 AI 财务建议
- 英语/多语言版 → 留学生群体
